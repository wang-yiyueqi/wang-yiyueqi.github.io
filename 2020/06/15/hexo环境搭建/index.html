github + hexo
总结hexo的搭建过程

需要安装

  • git
  • node.js

操作步骤

github方面

！ 如果第一次使用git的话，先注册账号之后，再进行以下操作。

  • 创建仓库
    仓库名：username.github.io
    ! 此处省略了配置SSH公钥和设置用户信息

hexo方面 搭建博客

！在进行博客搭建时，每次命令都需要在C:\Desktop\username.github.io\目录下！

  1. 运行cmd，在根目录利用npm命令安装hexo
    npm install -g hexo
  2. 在本地创建与仓库名同名的文件夹 username.github.io
  3. 打开文件夹，并且右键选择【Git bash here】
  4. 初始化建立独立博客所需要的所有文件夹 hexo init
  5. 安装依赖包 npm install
  6. 确保git部署 npm install hexo-deployer-git --save
  7. 本地查看

更换主题

知乎-hexo主题总结

  1. 找到喜欢主题的GitHub仓库

  2. 复制主题的HTTPS地址

  3. 执行git复制命令 将主题复制到博客中 git clone HTTPS地址 themes/指定主题名

  4. 修改整站配置文件：

    tips: 博客内的配置文件分为两种：一个是根目录下的_config.yml文件，另一个是主题下的_config.yml文件。
    这里修改的是根目录下的配置文件。

  5. 启用新下载的主题：
    将根目录下_config.yml文件中，将# Extension 默认主题landscape 修改为刚刚下载下来的主题名。

  6. 更新主题

将博客部署到github

  1. 复制仓库的SSH码
  2. 编辑根目录下的_config.yml文件
  3. 重新生成并部署

    如果部署失败，检查repository>setting>Deploy keys中有没有deploy权限
    Deploy keys>Add new时，不要忘记勾选上Allow write access

搭建完毕之后的踩坑

  1. 新建的md文件无法generate
    原因：根目录和主题的_config.yml文件中timezone设置值不同。
    解决办法：都统一修改成了 timezone: Asia/Tokyo

  2. 在主页面的menu栏，点击Home或者任意一个tab，都是跳转到404文件，而且访问的网址后面有/20%/
    原因：主题配置文件中menu下个各个子项目设置值，/后的空格多余，而导致找不到匹配文件
    解决办法: 删除空格。

修改后：

  1. 修改主题配置文件在本地启动服务预览有效果，deploy到github上无效果。
    原因：主题只是复制到了我的本地仓库中，部署到github上的博客里面不包含引用的主题仓库。
    解决办法： 将本地主题配置文件的修改部分剪切到了根目录的配置文件中，并且再次生成和部署。

  2. 让网站首页只显示每篇文章的部分内容，不要显示全部内容
    解决办法： 修改根目录下的主题文件。
